This is the cross reference table for many to many relationship between domain and community_categories tables.

This is the primary key for the table.

  • domain_pk1
  • community_categories_pk1

Foreign key constraints

    • domain_pk1
  2. domain_org_categories_fk2 (On Delete Cascade)

    This is a foreign key referencing the primary key of the community_categories table.

    • community_categories_pk1

Column name Data type Default value Value constraint Default constraint Identity? Nullable? Description
domain_pk1 id false false

This is the Foreign Key referencing domain table that is enforced in the code not at the database level. It is the domain that the organization catalog is linked to. A domain can have many organization catalogs.
community_categories_pk1 id false false

This is the Foreign Key referencing community_categories table that is enforced in the code not at the database level. It is the organization catalog that is linked to the domain. A organization catalog can be linked to multiple domains.
