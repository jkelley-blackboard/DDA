user_test_option

This table contains course user specific assessment settings information. It is a cross reference table for many to many relationship between course assessments (course_assessment table) and course users (course_users table).

multiple_attempts_ind char(1) false true

This boolean flag (Y/N/null) indicates if the assessment can be attempted multiple times, null means no exception setting for this field.
unlimited_attempts_ind char(1) false true

This boolean flag (Y/N/null) indicates if the assessment can be attempted unlimited times when multiple_attempts_ind sets to Y. N when multiple_attempts_ind is set to Y would indicate that the assessment is allowed for multiple attempts for the number of times set by attempt_count, null means no exception setting for this field.
attempt_count int false true

This is the number of attempts allowed when multiple_attempts_ind is set to Y and unlimited_attempts_ind to N, null means no exception setting for this field.
force_completion_ind char(1) false true

This boolean flag (Y/N/null) indicates if the assessment must be completed the first time it is launched, null means no exception setting for this field.
timer_completion char(1) false true

This is the enum value to show Continual (C) or Hardstop (H), null means no exception setting for this field.
time_limit int false true

This is the assessment time limit for completion in minutes.
soft_time_limit int false true

Unit in minutes. In the case of tests with a (time_limit), if this is null then it means there is no soft-time-limit and behaves same as Learn 9.1. If it's not null, user will receive an alert if the (soft_time_limit) expires. Depending on the value of (timer_completion), user will be forced to submit attempt ((timer_completion) is (H)) or be alerted again but can continue working without deadline ((timer_completion) is (C)) if the (time_limit) expires then. So (soft_time_limit) should be always less than (time_limit) if it's not null, this will be ensured and checked at REST API level.
