ai_playground_conversation

This table stores AI Playground conversation references per user. Actual message content is stored in LibreChat.

Primary key constraint

ai_pg_conv_pk

  • pk1

Foreign key constraints

  1. ai_pg_conv_fk1 (On Delete Cascade)
    • users_pk1

Indexes

  1. ai_pg_conv_if1
    • users_pk1
    • conversation_uid
    • context_type
  2. ai_pg_conv_if2
    • users_pk1
    • context_type
Column name Data type Default value Value constraint Default constraint Identity? Nullable? Description
pk1 id true false

Surrogate primary key of the AI Playground conversation table. Uniqueness of records is enforced by the 3 columns - users_pk1, conversation_uid, context_type.
users_pk1 id false false

Foreign key to users.pk1. Owner of the conversation.
conversation_uid varchar(100) false false

Conversation identifier returned by LibreChat.
title varchar(255) false false

User-visible title of the AI Playground conversation.
context_type varchar(50) false false

Context of the conversation (AI_PLAYGROUND, ADMIN_AI, etc).
model_name int 0 false false

The model used in the conversation, e.g. AMAZON_NOVA_MICRO, AMAZON_NOVA_LITE. This is represented as an int for easier querying and indexing, and the mapping of int to model name is maintained in the application code. The default 0 denotes None for backward compatibility.
created_at datetime NOW false false

Timestamp when the conversation was created.
last_modified_at datetime NOW false false

Timestamp of the last activity in the conversation.
