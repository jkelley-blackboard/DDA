DDA Database Overview

BB<deployment_id>

Primary database

Core application tables — courses, users, enrollments, grades, hierarchy, and tools. Most reporting and integration work starts here. Activity data is retained for approximately 180 days.

BB<deployment_id>_stats

Reporting and analytics database

Permanent copy of activity_accumulator with no 180-day truncation. ODS tables contain transformed data for activity reporting. Requires dblink for cross-database queries.

BB<deployment_id>_admin

System coordination database

Internal configuration and metadata. Rarely accessed for reporting or integrations — primarily used for system-level coordination.

BB<deployment_id>_cms_doc

Content Collection document store

Powers file storage and retrieval for the Content Collection. Not documented in schema packages and generally not used for reporting.

BB<deployment_id>_cms

Content Collection admin database

Manages metadata and configuration for the Content Collection. Rarely accessed and not typically relevant for reporting.

Replace BB<deployment_id> with your actual database prefix (e.g. BB5c1c67a3c99fc).
Legacy documentation may refer to this as bb_bb60 or BBLEARN.
