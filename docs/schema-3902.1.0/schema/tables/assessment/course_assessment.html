course_assessment

This table contains setting information on assessments within courses. It is a cross reference table for many to many relationship between courses (course_main table) and assessments (qti_asi_data table). Any changes here should also be done in course_assessment_user_vw under Assessments B2.

time_limit int false true

This is the assessment time limit for completion in minutes.
soft_time_limit int false true

Unit in minutes. In the case of tests with a (time_limit), if this is null then it means there is no soft-time-limit and behaves same as Learn 9.1. If it's not null, user will receive an alert if the (soft_time_limit) expires. Depending on the value of (timer_completion), user will be forced to submit attempt ((timer_completion) is (H)) or be alerted again but can continue working without deadline ((timer_completion) is (C)) if the (time_limit) expires then. So (soft_time_limit) should be always less than (time_limit) if it's not null, this will be ensured and checked at REST API level.

feedback_settings varchar(255) false true

Pipe (|) separated list of enabled student feedback options. List item are of the form -> state:feedbackOption1,feedbackOption2. Possible states are: as(After Submission), ashr(After submission, then hide results), osd(On Specific Date), add(After Due Date), aaed(After Availability End Date), aas(After Attempts are submitted), aag(After Attempts are graded), aiag(After individual grade is posted). Possible feedback options are: s(score), ua(useranswer), ca(correct), f(feedback), aa(allanswers), ia(incorrect).
