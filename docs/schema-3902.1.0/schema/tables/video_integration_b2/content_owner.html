content_owner

This table contains FKs to entities that can have VTBE/BbML associated with them. At most one FK may be non-null. When bulk delete ocurrs, such as when a course is deleted or when an attempt is deleted, the FK is set to null, and will be cleaned up in a different thread. So the state where all FKs, crsmain_pk1 excluded, are null is very short-lived and is required so that cleanup work which can caused delays is done asynchronously. As new entities are added which can have VTBE/BbML associated to them, a new FK column is added to this table. Note: This table allows tools with 1 or more VTBE fields to register as a content owner so that they can be associated generically to other objects in the database. As such, the table is not video integration specific and can be moved to core in the future.
