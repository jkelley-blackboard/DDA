domain_data_source

This is the cross reference table for many to many relationship between domain and data_source tables.

Primary key constraint

domain_data_source_pk

This is the primary key for the table.

  • domain_pk1
  • data_source_batch_uid
  • collection_type

Foreign key constraints

  1. domain_data_source_fk1 (On Delete Cascade)

    This is a foreign key referencing the primary key of the domain table.

    • domain_pk1
Column name Data type Default value Value constraint Default constraint Identity? Nullable? Description
domain_pk1 id false false

This is the Foreign Key referencing domain table that is enforced in the code not at the database level. It is the domain that the data source is linked to. A domain can have many data sources.
data_source_batch_uid nvarchar(256) false false

This is the Foreign Key referencing data_source table. A data source can be linked to multiple domains. This FK is not enforced.
collection_type char(1) false false

This is the type of the collection that the data source comes from. It can be course collection, module collection, user collection etc.
