asmt_upgrade_flags numeric 0 false false

After any import or restore operation the asmt_upgrade_version is set back to 0 and any new data re-upgraded. There are certain parts of the upgrade which do not have to run in this case (i.e. the upgrade is only required once per course, not after each import operation). This field is used to store those flags to avoid re-running pieces of the upgrade

course_view_option char(1) course_main_ck1
  • I

    Instructors can choose course view at anytime

  • U

    The course will use Ultra view, instructors can't change it

  • C

    The course will use Original view, instructors can't change it

 false true

This is course view option flag. It's different from ultra_status flag, this flag control if instructor can change the course view or change directly by admin, while ultra_status just indicate the current course view of this course is. Also, if the course ultra_status is in ULTRA_DECIDED state, then this flag will be ignored. If null, then the default value depends on the system-wide ultra setting: If ultra is ON then null is effectively 'I' If ultra is OFF then null is effectively 'C'

ultra_status char(1) ultra_status_enum
  • N

    No explicit decision has been made yet (for now treated the same as classic-decided)

  • C

    The classic user experience has explicitly been chosen

  • U

    The Ultra user experience has explicitly been chosen.

  • P

    The course is currently an Ultra course but in preview mode.

 false true

This is the flag indicating if the course is classic, ultra, or ultra preview. Note that updates to this field are controlled in java code so please do not blindly change it directly. Specifically you can only perform these transitions: N->C, N->U, N->P, C->U, C->P, P->U, P->C, U->NOTHING. You cannot go back once you pick U The NULL value, which represents classic, will be left for all the legacy courses and treated the same as classic
